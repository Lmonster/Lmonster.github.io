C++11 STL时间

  • 1m0nster

简介

在c++11之后，终于STL终于有了一个获取时间的库了—chrono。他可以获取精确的时间（到纳秒级），并且支持时间的计算。而且还保留了对于ctime的支持，可以从chrono的时间转换到std::time_t

在chrono下面有以下几个模块:

  • Clock
  • Time point
  • Duration
  • …… (一些c++20之后的特性，在这里不展开讨论)

本文的讨论仅限于c++11的范围，c++20 chrono库增加了很多新模块，但是不会在本文中展开

模块分解

Clock

clock提供了许多时钟的抽象。一个时钟包含的是开始时间点（epoch）和tick rate。比如常见的时钟就是从1970.1.1 00:00:00开始计时并且每一秒计时一次。STL提供了如下的时钟抽象

  • system_clock
    顾名思义，system_clock提供的是系统时钟。跟系统时间保持一致，是可以修改的。

  • steady_clock
    由于system_clock是一个系统时间的时钟，是可以由用户自由修改的，所以对于时间间隔的计算是存在不精确的可能性。而这个时钟就保证了时间的不可修改性，他所提供的时间是monotonic time（单调时间），是不能够篡改的。
    note:
    这个时间是从机器运行开始计时的，但是当机器关机的时候，原来的计时就会被置为0，也就是说，开机的时候计时会从0开始（也就是这个特性，可以用来精确的计算机器从最近一次开机到现在的运行的时间）

    这个类没有提供将时间点转换为std::time_t的功能，因为他计时的起点是机器开机时刻，并不是基于1970/01/01 00:00:00的，所以提供时间的转化是没有意义的。

  • high_resolution_clock

Time Point

这个模版类代表了基于clock的时间点 — 从计时起点，经过一个Duration，到某一个时间点。这个类跟时钟紧密相关，不同时钟（时间坐标系不同）的时间点不能直接运算。
获取时间间隔(Duration)的方法是：time_since_epoch()
