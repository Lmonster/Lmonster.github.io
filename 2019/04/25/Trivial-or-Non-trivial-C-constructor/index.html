Trivial or Non-trivial C++ constructor

  • 1m0nster
  • 0 Minutes
  • April 25, 2019

Trivial定义为“微不足道的”，所以trivial constructor的意思就是没有任何作用的构造函数，就是说这个构造函数并没有不会做任何事情。反之，non-trivial constructor就是会做一些事情的构造函数。

default constructor可以是trivial的，也可以是non-trivial的。那么如何做出这两者之间的区分呢？

  • 如果用户自定义了default constructor的话，那么这个构造函数就是non-trivial构造函数
  • 如果是编译器默认定义的构造函数（用户并没有自定义默认构造函数），满足下面四个条件之一编译器会生成一个non-trivial 默认构造函数：
    1. 类中有存在默认构造函数的成员变量
    2. 类所继承的父类中存在默认构造函数
    3. 类中有虚函数
    4. 类继承了虚基类

除了上述四种情况，其他情况下编辑器在用户没有自定义默认构造函数时，会合成一个trivial-constructor。其实并不会生成，因为这个构造的本身并没有任何意义。
由上可知，并不是所有的用户没有自己生成默认构造函数，编译器就会为这个类生成默认构造函数。
