## explicit存在的意义

\1. 减少歧义，增强代码的可读性。比如：

一个类定义了一个如下的构造函数:

`

class ClassType {

ClassType(int) {

}

};

`

就可以用下面的方式构造一个实例

`

ClassType instance = 1;

`

对于不熟悉这个类的人而言，很容易造成误解，这个ClassType到底是一个如typedef int ClassType的定义还是ClassType是一个类。

\2. 解决一种被叫做Schwarz error的错误

比如：

std::cin之前的版本定义了如下的conversion

operator int(); 那么就会有这样的错误出现

cin<<aIntVariable;

这是一个很明显的错误。也许使用者是想用cout，但是却错用了cin。

不过，在编译的时候并不会报错。为什么？因为有一个conversion会把cin变为int，那么<<j就是左移的意思了。
